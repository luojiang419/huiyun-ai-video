## 一、架构与代码质量

### 1.1 重复代码：`_loadSystemPrompt` 方法
- **位置**：[storyboard_screen.dart:92](flutter_grsai_video_gen/lib/screens/storyboard_screen.dart#L92) 和 [film_workshop_screen.dart:81](flutter_grsai_video_gen/lib/screens/film_workshop_screen.dart#L81)
- **问题**：两处代码几乎完全相同，从同一路径读取同一 `storyboard_system_prompt.txt` 文件，逻辑完全重复。
- **建议**：提取到 `FilmWorkshopStorageService` 或新建 `PromptStorageService`，统一管理。

### 1.2 重复代码：`_loadState` 方法
- **位置**：[film_workshop_screen.dart:103](flutter_grsai_video_gen/lib/screens/film_workshop_screen.dart#L103)
- **问题**：`isGenerating` 和非 `isGenerating` 两条分支中，参考图片和备注同步代码完全相同（约 20 行重复）。
- **建议**：提取为私有方法 `_applyReferenceImagesFromState(state)`。

### 1.3 `addAsset` 空实现
- **位置**：[film_project_provider.dart:243](flutter_grsai_video_gen/lib/providers/film_project_provider.dart#L243)
- **问题**：方法体为空（含大段注释），调用方添加资产后 UI 不会刷新。
- **建议**：调用 `ref.read(assetProvider.notifier).reload()` 触发 AssetProvider 刷新，或直接删除该方法改为在 UI 层直接调用 AssetProvider。

### 1.4 生产环境禁用 HTTPS 证书校验（安全漏洞）
- **位置**：[api_service.dart:48](flutter_grsai_video_gen/lib/services/api_service.dart#L48)
- **问题**：`badCertificateCallback` 恒返回 `true`，在生产版本中存在中间人攻击风险。
- **建议**：用 `kDebugMode` 条件包裹，Release 环境使用正常证书校验。

## 二、用户体验（UX）

### 2.1 剧本字数超限无视觉警告
- **位置**：[storyboard_screen.dart:340](flutter_grsai_video_gen/lib/screens/storyboard_screen.dart#L340)
- **问题**：显示 `当前字数：xxx / 800`，但超出 800 字时字色不变，用户无感知。
- **建议**：超 800 字时将字数文字变为红色，可选添加轻微震动提示。

### 2.2 批量生成无总进度显示
- **位置**：[generate_logic_service.dart:56](flutter_grsai_video_gen/lib/services/generate_logic_service.dart#L56)
- **问题**：批量生成时助手消息显示 `批量生成中 (0/N)`，但 UI 层无进度条，用户体验差。
- **建议**：在 GenerateScreen 的助手消息卡片中增加线性进度条，根据 completed/total 计算进度。

### 2.3 失败镜头无单独重试按钮
- **位置**：[film_workshop_screen.dart](flutter_grsai_video_gen/lib/screens/film_workshop_screen.dart)
- **问题**：镜头生成失败后，用户必须重新全量生成，无法单独重试失败项。
- **建议**：在状态为 `failed` 的镜头卡片上增加「重试」按钮，单独触发该镜头的生成。

### 2.4 使用已废弃的 `RawKeyboardListener`
- **位置**：[generate_screen.dart:160](flutter_grsai_video_gen/lib/screens/generate_screen.dart#L160)
- **问题**：`RawKeyboardListener` 在 Flutter 3.x 中已废弃，会产生 deprecation warning。
- **建议**：替换为 `KeyboardListener`（`HardwareKeyboard` API）。

### 2.5 最小窗口尺寸限制过严
- **位置**：[main.dart:21](flutter_grsai_video_gen/lib/main.dart#L21)
- **问题**：`minimumSize: Size(1300, 760)` 对低分辨率屏幕（如 1280x720 笔记本）不友好。
- **建议**：降低至 `Size(1100, 680)` 或增加响应式布局支持。

### 2.6 资产图片使用 `Image.network` 加载本地路径
- **位置**：[storyboard_screen.dart:304](flutter_grsai_video_gen/lib/screens/storyboard_screen.dart#L304)
- **问题**：`Image.network(asset['imagePath'])` 无法加载本地文件路径，导致资产预览图不显示。
- **建议**：改为 `Image.file(File(asset['imagePath']))`，并加 `errorBuilder` 容错。

## 三、性能优化

### 3.1 每次击键触发磁盘写入（缺少防抖）
- **位置**：[storyboard_screen.dart:48](flutter_grsai_video_gen/lib/screens/storyboard_screen.dart#L48)
- **问题**：`_projectNameController.addListener(_saveState)` 等四个监听器在每次字符输入时都触发 `saveStoryboardState`（磁盘 IO）。高频写入浪费 IO，可能造成轻微卡顿。
- **建议**：引入 `debounce`（延迟 800ms），使用 `Timer` 实现或引入 `rxdart` 包。

### 3.2 批量生成异常时 Map 内存泄漏
- **位置**：[generate_logic_service.dart:128](flutter_grsai_video_gen/lib/services/generate_logic_service.dart#L128)
- **问题**：`catch` 块中只递增 `_batchCompleted` 计数，但未在全部失败时清理 `_batchResults` 和 `_batchCompleted`，存在 Map 条目泄漏。
- **建议**：在 catch 块中判断 `completed >= totalBatch` 时同样执行清理逻辑。

### 3.3 参考图 Base64 存入状态 JSON 导致文件体积过大
- **位置**：[film_workshop_storage_service.dart:8](flutter_grsai_video_gen/lib/services/film_workshop_storage_service.dart#L8)
- **问题**：`saveWorkshopState` 将参考图片 base64 字符串直接序列化进 JSON，单张图片可达 500KB+，多张图片时状态文件可达数 MB，加载/保存慢。
- **建议**：状态文件只存图片路径，图片内容在使用时按需读取。

## 四、功能扩展建议

### 4.1 分镜导出功能
- **现状**：分镜生成后只能在软件内查看，无法导出。
- **建议**：增加「导出分镜」按钮，支持导出为 PDF（图+文字）或图片网格，方便导演/制片沟通。

### 4.2 生成页提示词历史记录
- **现状**：`GenerateScreen` 无提示词历史，用户重复输入麻烦。
- **建议**：在输入框下方增加最近 10 条提示词历史（本地 JSON 存储），点击快速填入。

### 4.3 工作台镜头拖拽排序
- **现状**：`FilmWorkshopScreen` 中镜头列表固定顺序，无法手动调整。
- **建议**：使用 `ReorderableListView` 替换当前列表，支持拖拽调整镜头顺序。

### 4.4 API 配置连通性测试按钮
- **现状**：`ApiConfigScreen` 保存配置后无法验证是否能正常调用。
- **建议**：增加「测试连接」按钮，发送一次简单 chat 请求，显示成功/失败及延迟。
