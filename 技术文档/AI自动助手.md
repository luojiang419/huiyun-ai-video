# 影视版 - 图片生成页面 AI自动助手 功能优化方案

---

## 一、功能概述

将现有图片生成页面的「AI润色模式」升级为「AI自动助手」模式，实现从简单提示词润色 → 完整对话式AI生成助手的跨越，并加入**技能库自学习系统**。

### 现有功能
- 右上角 "AI" 按钮切换 `_isAiPolishMode`
- 开启后，生成前自动调用 `autoPolishPrompt()` 润色提示词
- 润色后直接提交生成，无对话交互

### 升级目标
- 开启AI模式后，进入**对话式生成**体验
- AI助手自动学习所有图片生成模型的提交方式与参数
- AI回复包含**可点击选项**，用户可直接点选操作
- 以自然对话引导用户描述需求，确认需求后自动润色提示词并提交生成
- 生成完成后**自动询问满意度**，满意则**记录为技能**存入技能库
- 后续类似题材**自动调用技能库**中的成功经验
- 内置丰富的**专业影像提示词技能**，让AI助手更专业

### 2.2 可点击选项设计规范

AI助手的每条回复都可以携带可点击的选项按钮，用户无需手动输入，直接点击即可：

| 选项类型 | 样式 | 触发行为 |
|---------|------|---------|
| 快捷分类 | 带图标的胶囊按钮 | 切换到该分类的技能推荐 |
| 确认操作 | 绿色实心按钮 | 执行对应操作（生成/保存等） |
| 修改操作 | 蓝色描边按钮 | 弹出修改面板或进入修改对话 |
| 风格选择 | 标签式按钮（可多选） | 切换提示词风格 |
| 参数调整 | 小标签按钮 | 弹出参数选择器 |
| 技能调用 | 卡片式按钮 | 加载该技能的完整配置 |

## 五、核心模块详细设计

### 5.1 技能库服务 (`skill_service.dart`)

```dart
class SkillService {
  // 技能库根目录：data/Skills/

  /// 初始化技能库目录结构
  Future<void> initSkillDirectories()

  /// 加载技能索引
  Future<SkillIndex> loadIndex()

  /// 加载所有技能（内置 + 用户自学习）
  Future<List<Skill>> loadAllSkills()

  /// 根据用户需求匹配最相关的技能
  Future<List<SkillMatch>> matchSkills(String userInput)

  /// 获取指定分类下的所有技能
  Future<List<Skill>> getSkillsByCategory(String categoryId)

  /// 保存用户自学习技能
  Future<Skill> saveUserSkill(UserSkillSaveRequest request)

  /// 记录技能使用次数
  Future<void> incrementUsageCount(String skillId)

  /// 更新技能索引
  Future<void> updateIndex()

  /// 删除用户技能
  Future<void> deleteUserSkill(String skillId)

  /// 技能匹配算法：
  /// 1. 关键词匹配：用户输入 vs 技能tags
  /// 2. 语义相似度：通过AI判断需求与技能的关联度
  /// 3. 使用频率加权：常用技能优先
  /// 4. 评分加权：高评分技能优先
  /// 返回 top-3 匹配结果，附带匹配度分数
}

class SkillMatch {
  final Skill skill;
  final double relevanceScore;  // 0.0 ~ 1.0
  final String matchedReason;   // 匹配原因说明
}
```

### 5.2 AI助手服务 (`ai_assistant_service.dart`)

```dart

  enum AssistantPhase {

    confirming,    // 确认阶段
    generating,    // 生成阶段
    reviewing,     // 结果审核 + 满意度询问
    saving_skill,  // 技能保存阶段

### 7.2 技能自动推荐算法

当用户提出新需求时，系统自动匹配技能：

```dart
Future<List<SkillMatch>> matchSkills(String userInput) async {
  final allSkills = await loadAllSkills();
  final matches = <SkillMatch>[];

  for (final skill in allSkills) {
    double score = 0.0;

    // 1. 关键词匹配（权重40%）
    for (final tag in skill.tags) {
      if (userInput.contains(tag)) {
        score += 0.4 / skill.tags.length;
      }
    }

    // 2. 分类匹配（权重20%）
    if (skill.category.toLowerCase().contains(
      userInput.toLowerCase())) {
      score += 0.2;
    }

    // 3. 使用频率加权（权重15%）
    score += 0.15 * (skill.usageCount > 0
      ? (1.0 / skill.usageCount).clamp(0.0, 1.0)
      : 0.5);

    // 4. 评分加权（权重15%）
    score += 0.15 * (skill.rating / 5.0);

    // 5. 用户自学习技能加权（权重10%）
    if (skill.source == 'user') {
      score += 0.1;
    }

    if (score > 0.3) {  // 最低匹配阈值
      matches.add(SkillMatch(
        skill: skill,
        relevanceScore: score.clamp(0.0, 1.0),
        matchedReason: _buildMatchReason(skill, userInput),
      ));
    }
  }

  // 返回匹配度最高的3个技能
  matches.sort((a, b) => b.relevanceScore.compareTo(a.relevanceScore));
  return matches.take(3).toList();
}
```

## 九、与现有功能的兼容性

### 9.1 模式切换
- AI模式开启时：隐藏原有参数栏（模型/比例/尺寸/批量），由AI自动选择
- AI模式关闭时：恢复原有手动参数选择界面
- 模式切换不丢失对话历史

### 9.2 会话管理
- AI助手的对话历史存储在现有 Session 模型中
- 每个会话独立的AI助手对话上下文
- 支持新建/切换/删除会话

### 9.3 API调用
- AI对话使用 `type: "chat"` 的API配置（如豆包、GLM4.5）
- 图片生成使用 `type: "image"` 的API配置（如 gemini-3-pro）
- 两个API独立工作，互不干扰

### 9.4 生成逻辑复用
- AI助手确认后调用现有 `GenerateLogicService.generate()` 方法
- 复用现有的进度跟踪、错误处理、图片下载保存逻辑

### 9.5 技能库存储
- 技能文件存储在 `data/Skills/` 目录下
- 用户自学习技能与内置技能分离存放（`user/` vs `builtin/`）
- 技能索引文件 `index.json` 维护全局快速检索

## 十二、注意事项

1. **AI回复格式**：需严格使用JSON格式输出计划和选项，方便程序解析，同时保持自然语言回复给用户
2. **可点击选项解析**：AI返回的JSON中 `options` 数组需要可靠解析，做好降级处理（解析失败时只显示文本）
3. **技能匹配效率**：技能库加载时构建索引缓存，匹配时优先查索引而非遍历所有文件
4. **自学习质量**：用户保存技能时自动提取关键信息（tags、模板、规则），必要时通过AI辅助提取
5. **技能库隔离**：内置技能只读不可删除，用户技能可编辑和删除
6. **错误兜底**：AI对话API异常时，提供降级方案（如提示用户切回手动模式）
7. **流式响应**：AI对话使用流式输出（`chatStream`），提升用户体验
8. **上下文窗口**：对话历史过长时自动截断早期消息，保留最近10轮对话
9. **技能库同步**：编译新版本时内置技能文件随软件分发，用户技能独立存储不丢失
10. **多模型切换**：用户在对话中可指定使用特定模型，AI应识别并遵从
